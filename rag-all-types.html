RAG Fusion
Generates multiple query angles, retrieves for each, then fuses results using Reciprocal Rank Fusion.
Best coverage Multi-angle 4× retrieval cost
WHY USE A single query often misses the best documents due to wording mismatch. By generating 4 perspectives and fusing results by overlap score, you get broader, more balanced context — especially for research-type or vague questions.

Speculative RAG
Small model drafts fast. Large model verifies. Speed + accuracy without full cost.
Fast + accurate Production-grade Two LLM calls
WHY USE A cheap fast model handles 80% of traffic correctly. The expensive model only verifies — so average latency and cost stay low even at scale. Best for production apps with response-time SLAs.

Adaptive RAG
Dynamically chooses the right retrieval strategy per query — no-retrieval, single-step, or multi-step.
Smart routing Query-aware Efficient
WHY USE Not every question needs retrieval. Asking "what is 2+2?" doesn't need a vector search. Adaptive RAG classifies the query complexity first, then picks the cheapest strategy that can answer it correctly — saving cost and latency on simple queries.

Modular RAG
Breaks the RAG pipeline into swappable components — each module is independent and replaceable.
Composable Flexible Production-ready
WHY USE Real production systems need flexibility — swap the embedding model, change the retriever, update the re-ranker, or plug in a new LLM without rebuilding the whole pipeline. Modular RAG is an architecture pattern, not a technique.

IDEA 01
SaaS AI Product Platform
A company builds one RAG framework and lets enterprise clients plug in their own retriever, LLM, and re-ranker. Each client's deployment is identical in structure but customised per module.
Multi-tenant architecture
IDEA 02
Regulated Industry Bot
A fintech company swaps the LLM module for a locally-hosted model for data privacy, while keeping the same retrieval + re-ranking modules. No full rebuild — just module swap.
Compliance-driven swap
IDEA 03
A/B Testing RAG Quality
ML team runs experiments: swap the embedding model module (OpenAI vs Cohere vs BGE) and measure answer quality independently — without changing retrieval logic, re-ranker, or LLM module.
Experiment-friendly

Agentic RAG
The LLM becomes an agent — it plans, decides which tools to call, and iterates until it has enough information to answer.
Multi-step reasoning Tool use Most powerful
WHY USE Some questions can't be answered in one retrieval step. Agentic RAG lets the LLM reason about what it still needs to know, call tools (search, calculator, APIs, databases), and iterate — just like a human researcher would.

Graph RAG
Retrieves from a knowledge graph of entities and relationships, not just text chunks.
Relationship-aware Deep connections Complex queries
WHY USE Vector search finds semantically similar chunks, but it can't answer "how are these two entities connected?" or "what is the chain of effects from event A to outcome B?" Graph RAG traverses explicit relationships between entities — perfect for interconnected knowledge domains.

IDEA 01
Drug Interaction Checker
Medical KB as a graph: Drug → treats → Disease, Drug → inhibits → Enzyme, Enzyme → metabolises → Drug. Query "Can patient take Drug A + Drug B?" traverses the interaction graph for conflicts.
Relationship chains matter
IDEA 02
Corporate Knowledge Map
Graph of: Employee → works on → Project, Project → depends on → Team, Team → reports to → Manager. Query "who should I talk to about Project X?" traverses the org graph to find the right person.
Org connections hidden in docs
IDEA 03
Fraud Detection Explainer
Financial transactions as a graph. Query "Is this account suspicious?" — Graph RAG traverses: Account → transacts with → Account → linked to → flagged entity, surfacing the chain of suspicion.
Multi-hop reasoning

Recommended Learning Path
Build in this order — each step teaches you why the next one exists.
①
Basic RAG
Build a handbook chatbot with LangChain + FAISS. Feel where it fails (bad retrieval → bad answers).
②
Corrective RAG
Add a quality grader + web search fallback. See how a self-healing pipeline prevents outdated answers.
③
RAG Fusion
Add query rewriting + RRF. Notice how complex questions suddenly return much better context.
④
Self-RAG
Add self-grading loops. See hallucinations caught and retried. Now you're ready for high-stakes production.
⑤
Agentic / Graph RAG
Build a multi-tool agent or knowledge graph. You'll have all the context to understand why these are powerful and expensive.
